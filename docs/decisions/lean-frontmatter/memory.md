# Decision: Lean frontmatter

## Context

Previous version had created_by, updated_by, created, updated, type, status, priority
and many other fields. Bloated every file.

## Decision

Seven fields only: title, summary, kind, tags, refs, by, at.

- title: human readable name
- summary: one sentence for browsing (generated by Claude at write time)
- kind: folder category
- tags: filtering labels (slugified automatically)
- refs: bidirectional links to other memory_file paths
- by: last editor
- at: last modified (git handles full history)

## Why

- Every field costs context when loaded into cache
- "by" replaces created_by/updated_by — git blame has the full history
- "at" replaces created/updated — git log has the timestamps
- No "type" — kind IS the type
- No "status" — that's a workflow concern, not a memory concern

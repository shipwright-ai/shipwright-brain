Tag with severity and area. Check brain://overview for existing tags first — reuse when they fit, create new ones when they don't.

Write as a checklist — each fix/investigation step is a checkbox:

> Observed: what's broken
> Expected: what should happen
> Context: how it was discovered, date

- [ ] Reproduce the issue
- [ ] Identify root cause
- [ ] Implement fix
- [ ] Verify fix
